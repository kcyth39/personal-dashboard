# Personal Dashboard プロジェクト仕様書 (Project Specification)

## 1. プロジェクト概要 (Overview)
「起きてブラウザを開くだけ」でその日の主要な情報を一覧できる、個人専用のモダンな朝礼用ダッシュボード画面です。毎日のルーティンをスムーズにするための情報や素早いアクセス手段をひとまとめにし、Macのバックグラウンドで常に動き続けるように構築されています。

## 2. 使用技術 (Tech Stack)
- **コアフレームワーク:** React 19 (Vite)
- **スタイリング:** Tailwind CSS v4 (vanilla CSSによる拡張グラスモーフィズムを含む)
- **アイコン:** `react-icons` (FaRegNewspaper, FaRegCalendarAlt, FaLink, FaLightbulb など)
- **常駐環境:** PM2 (Node.jsのプロセスマネージャーによるバックグラウンドデーモン化)

## 3. UI / デザイン方針 (Design Aesthetics)
- **グラスモーフィズムデザイン:** `glass-panel` クラスを利用した、背景がすりガラスのように透けるモダンなUI。
- **ダイナミックなグラデーション:** 文字色や背景色などに美しいグラデーション (`bg-gradient-to-r`) を用いたリッチな表現。
- **ダークモード対応:** OSのテーマ設定等に連動して黒背景ベースに切り替わるように、全体にわたって `dark:` プレフィックスを活用したパレット適応。
- **レスポンシブデザイン:** Viteのデフォルト機能とTailwindのグリッドレイアウトを用いて、画面幅に応じてレイアウトが崩れないよう `grid-cols-1 md:grid-cols-12` などを指定。

## 4. コンポーネント構成 (Components Setup)

現在、ダッシュボードは以下のメイン領域と4つの主要なカードコンポーネントから構成されています（コンポーネントごとに独立して拡張可能）。

### 1) メインヘッダー (`App.jsx` 内)
*   **内容:** 「Good Morning, Shige!」というパーソナライズされた挨拶と、現在の時刻をリアルタイム（毎秒更新 `setInterval`）で表示させる。
*   **用途:** 朝一番のモチベーションを高める起点。

### 2) NewsCard (`components/NewsCard.jsx`)
*   **機能:** 日々の重要ニュースやマーケット情報などを表示する大きな枠組み。
*   **現状:** UIのモックアップ段階（「Loading news...」プレースホルダー）。
*   **拡張構想:** 外部のニュースAPIやRSSなどをフェッチして動的にニュースリストを描画する。

### 3) CalendarCard (`components/CalendarCard.jsx`)
*   **機能:** 当日の日付、曜日を表示し、一日のスケジュールを伝えるカード。
*   **現状:** 標準の `Date` オブジェクトから当日の日付を取得して表示。予定部分は「No events scheduled.」の固定テキスト。
*   **拡張構想:** Google Calendar APIとの連携や、固有のJSONファイルからの予定読み込み。

### 4) LinksCard (`components/LinksCard.jsx`)
*   **機能:** 普段よく使うウェブサイトやポートフォリオ、メール画面などへのボタンが並んだランチャー機能。
*   **現状:** 配列データ（Workspace, Portfolio, Mail）でハードコードされたリンク集を描画。
*   **拡張構想:** 外出しの設定ファイル(`.json`)から設定を読み込ませることで、コードをいじらずにリンクの追加・削除を可能にする。

### 5) TipsCard (`components/TipsCard.jsx`)
*   **機能:** プログラミングやMacの小技、日常の健康といった「ちょっとした気付き」を日替わりで表示するカード。
*   **現状:** コンポーネント内の配列 (`tips`) に登録された複数のTipsから、当日の日付（`new Date().getDate()`）を使ってローテーションで1つを表示する簡易ロジック。
*   **拡張構想:** 大量のTipsデータを持たせた外部データからの読み込み。

## 5. 運用と実行に関する仕様 (Operations)
*   **ローカルサーバーの起動:** `npm run dev` （ポート: 5173 を基本とし、使用済みの場合は自動回避）
*   **バックグラウンド常駐:** `npx pm2 start npm --name "personal-dashboard" -- run dev` を用いて、ターミナルを閉じても常にバックグラウンドでシステムを保つ。
*   **エラー管理とログ:** PM2によって標準出力およびエラーは `/Users/shige/.pm2/logs/` 内に監視・記録される。

## 6. 今後の開発ロードマップと拡張方針
*   ハードコードされている情報をJSONやデータベース等の「設定ファイル群」と「システム本体」で分離し、**バージョンアップや機能拡張の際にも個人のデータが上書きされない設計（No-codeでデータ設定が可能な状態）**へとリファクタリングすること。
*   `eslint` を用いたコード解析によるエラーの未然防止を徹底すること。
*   開発ログを `diary.md` に残し、非エンジニア向けの情報発信を継続すること。
